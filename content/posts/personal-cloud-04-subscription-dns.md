订阅体系可以拆成两件事：

1. **拉配置**：客户端访问订阅 URL，下载节点列表；
2. **连节点**：按列表里的服务器地址真正建立代理连接。

二者经常共用「我的域名」，但 **Cloudflare 是否橙云代理**、**是否应走本地规则代理**，答案完全不同。

## 拉订阅 vs 连节点

| | 拉订阅 `/sub/...` | 连节点 |
|--|-------------------|--------|
| 协议 | HTTPS 网页/API | VLESS/Reality 等 |
| 域名例子 | 面板域名 | `区域-auto.example.com` 或 IP |
| CF 橙云 | 可以（走 HTTP 反代） | **通常不行** |
| 延迟敏感 | 低（偶尔更新） | 高（持续流量） |

节点入口若误开橙云，客户端会把代理握手打到 Cloudflare HTTP 边缘，结果多半是超时或异常。

## 「自动切换」域名在做什么

对「一组可替换后端」可以维护一个专用域名，例如概念上的 `jp-auto.example.com`：

```text
定时任务检查：
  - 后端健康（端口 / 探测）
  - 云流量配额是否接近阈值
  - 优先级策略

需要切换时：
  调用 DNS API，把 A 记录改到新后端 IP
  （TTL 宜短，便于收敛）
```

客户端订阅里 **服务器字段写域名**，不必每次改订阅内容。稳态延迟与写死 IP 基本相同，只是多一次可缓存的 DNS 解析。

## Clash / Mihomo 规则：访问自家域名别套代理

默认「非国内走代理」规则集时，`*.你的域名` 往往被判成境外站点，于是：

```text
浏览器 → 代理节点 → 再访问 面板/网盘/状态页
```

这会绕远，还可能在部分链路上更不稳。应在规则前部显式：

```yaml
rules:
  - DOMAIN-SUFFIX,example.com,DIRECT
  - DOMAIN,example.com,DIRECT
  # ... 其余规则
```

注意：这是 **规则分流**，不是「节点服务器字段别用域名」。节点仍可用域名做自动切换；**浏览自己网站时直连**即可。

## 面板与入口

- 订阅面板放在控制中枢，经 SNI 或 Tunnel 对外。
- 主机列表里同时存在「固定 IP 节点」与「自动切换域名节点」很常见。
- 禁用/启用某后端时，除了面板开关，自动切换任务的健康检查也要一致，避免 DNS 指到已下线机器。

## 小结

**订阅 URL** 与 **节点地址** 分层设计；自动切换靠 **短 TTL + DNS API**；客户端规则里 **自有域名 DIRECT**。下一篇专门写网盘：为什么隧道期会慢，直连后又注意什么。
